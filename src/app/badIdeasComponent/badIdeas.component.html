Controllers

Controllers are awesomeeeee!!!! Make sure you use controllers for everything. Every controller must meet the following criteria:
  • It must be at least 300 lines of code long.
  • It must make use of $on / $emit or $broadcast exactly six times.
  • It must modify the DOM directly, ideally using JQuery.
  • Have at least twenty references to $scope, and at least one reference to $rootScope.
  • Keep multiple models in the same controller, which are almost named the same but do completely different things.
  • Paste the controller two times in different folders, in order to make sure the code doesn't get accidentally deleted.
  • Ensure each controller makes at least five $httpBackend calls.
  • If you need to make the same $htpBackend call twice, paste it again.
  • If you have a piece of code you need executed with every $digest (such as an HTTP call), place it in a $watch.

Directives

  • Make sure to create a directive that loads your application via a CSS Class tag. This is suggested by the angular team here
  • Add a nice loading bar directive to your application which is displayed once your application has finished loading.

Services

  • For optimal reusability, if ever need to write a service (why?) make sure it calls at least another service which in turn calls another service to retrieve information.

Build

  • Make sure the total lines of gulp code exceeds the total lines of application code at all times.
  • Make sure the production build is larger than the development build.
